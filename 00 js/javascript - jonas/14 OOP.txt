# ---------------------------------- 217. What is Object-Oriented Programming? #
üìå NOTES
    - ‚úÖ object-oriented programming or OOP is a programming paradigm that is based on the concept of objects.
    - ‚úÖ paradigm (the style of the code, how we organize code and how we write the code)
    - ‚úÖ we use objects to describe aspects of the real world like (a user / to-do-list-item / HTML components / some kind of data structure).
    - ‚úÖ objects can contain data(properties) and its corresponding behaviour(methods).
    - ‚úÖ we can say that we pack data and its corresponding behaviour into one big block which is called object.
    - ‚úÖ in OOP, objects are like self contained block of code which act as a small application on its own.
    - ‚úÖ objects are building blocks of the applications, and interact with one another.
    - ‚úÖ interactions happens through a public interface (API): methods that the code outside  of the object can access and use to communicate with the object.
    - ‚úÖ OOP was developed with the goal of organizing code, to make it more flexible and easier to maintain (to avoid spagatti code).
    - ‚úÖ OOP was was most widely used programming paradigm in large scale software companies.
    - ‚úÖ remember OOP was not only the programming paradigm in coding, but we do have functional programming paradigm which is also becoming very popular.


üõ©Ô∏èüõ©Ô∏è Class and Instance (traditional OOP)
    - ‚úÖ in traditional OOP, we use Classes to create new objects.
    - ‚úÖ Class is a like a blue-print from which new object are created based on the rules described in the class.
    - ‚úÖ so from a single class, we can create 'n' numbers of objects.
    - ‚úÖ each object can have different data in them but all will share the same functionality in the class.


üõ©Ô∏èüõ©Ô∏è Four Fundamental OOP Principles
    - ‚úÖ there are 4 Fundamental Principles that guide us toward a good class implementation.
    - ‚úÖ Abstraction, Encapsulation, Inheritance and Polymorphism.


üõ©Ô∏èüõ©Ô∏è Abstraction
    - ‚úÖ ignoring or hiding details that dont matter, allowing us to get an overview perspective of the thing we are implementing, instead of messing with details that dont really matter to our implementation.
    - ‚úÖ real example for Abstraction is our mobile phone because we dont get all the details about our phone (verifyTemp(), verifyVol(), rearCameraOn(), rearCameraOff()).
    - ‚úÖ but we get the basic required functioanlities (homeBtn(), volumeBtn(), Screen(), etc).
    - ‚úÖ so the remaining functionalities are abstracted from us.
    - ‚úÖ some real world examples of Abstraction are Car, TV Remote ,E-Commerce Platform, etc.
    - ‚úÖ as a user we dont need some functionalities so that can be abstracted from the users.


üõ©Ô∏èüõ©Ô∏è Encapsulation
    - ‚úÖ keeping properties and methods private inside the class, so they are not accessisble from outside the class.
