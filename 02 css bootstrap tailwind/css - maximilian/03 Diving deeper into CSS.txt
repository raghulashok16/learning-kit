# -------------- HTML Refresher: Block-level vs Inline Elements -------------- #
âœ… 1. Block-level elements are rendered as a block and hence take up all the available horizontal space.
âœ… 2. You can set margin-top and margin-bottom and two block-level elements will render in two different lines.
âœ… 3. <div> , <section> , <article> , <nav>  but also <h1> , <h2>  etc and <p> are block-level-elements.
âœ… 4. Inline elements on the other hand only take up the space they require to fit their content in.
âœ… 5. Hence two inline-elements will fit into the same line (as long as the combined content doesn't take up the entire space in which case a line break would be added).
âœ… 6. They also use the box-model you learned about but margin-top  and margin-bottom  have no effect on the element.
âœ… 7. padding-top  and padding-bottom  also have a different effect.
âœ… 8. They don't push the adjacent content away but they will do so with the element border. 
âœ… 9. setting a width  or height  on an inline element also has no effect.
âœ… 10. The width and height is auto to take as much space as required by the content.
âœ… 11. Logically, this makes sense since you don't want your inline elements to destroy your multi-line text-layout.
âœ… 12. If you want to do so or need both block-level and inline behavior, you can set display: inline-block  to merge behaviors.
âœ… 13. Some example elements are: <a> , <span> , <img>.
